JavaScript

Для вставки JavaScript кода на страницу, как правило, используют специальный тег <script> У этого тега есть 6 атрибутов:

  • async(необязательный)
  • charset(необязательный)
  • defer(необязательный)
  • language(устарел)
  • src(необязательный)
  • type(необязательный)

Некоторые браузеры, в их числе и Internet Explorer, очень любят закрывающие теги и могут неправильно обрабатывать одинарные.

поэтому вместо:

<script type="text/javascript" src="example.js"/>

будем использовать данный вид записи:

<script type="text/javascript" src="example.js"></script>

DOM

Кроме того, если тегу <script> назначен атрибут src, то скрипт загрузится, но код между открывающим и закрывающим тегом не выполнится.

Такой код работать не будет

<script type="text/javascript" src="example.js">
    alert("Hello!");
</script>

<script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js">
</script>

так же можно подключить скрипты с внешних серверов если известен полный url адрес этих серверов

Не очень удачный способ размещения тега <script>!

Здесь показан правильный способ размещения тегов скриптов на странице.

<!DOCTYPE html>
<html>
<head>
    <title>Example</title>
    <script type="text/javascript" src="examplel.js"></script>
    <script type="text/javascript" src="example2.js"></script>
</head>
<body>
    <!--контент-->
</body>
</html>

<!DOCTYPE html>
<html>
<head>
  <title>ExampleHTMLPage</title>
  <script type="text/javascript" defer src="examplel.js"></script>
  <script type="text/javascript" defer src="example2.js"></script>
</head>
<body>
  <!--контент-->
</body>
</html>

Отложенные сценарии

Атрибут defer указывает браузеру, что скрипт нужно начать загружать немедленно, но запуск сценария, который содержится в данном скрипте следует отложить.

Данный скрипт не начнет выполняться пока браузер не встретит закрывающий тег </html>

...данный атрибут предназначен для загрузки внешних скриптов и его наличие у встроенных сценариев браузеры игнорируют.

<!DOCTYPE html>
<html>
<head>
<title>ExampleHTMLPage</title>
  <script type="text/javascript" defer src="singular.js"></script>
   </head>
<body>
  <script defer type="text/javascript">
    alert("Привет из встроенного скрипта!");
  </script>
  <!--контент-->
</body>
</html>

Асинхронные сценарии

Атрибут async появился в HTML5 и сообщает браузеру, что скрипт, также как и в случае с атрибутом defer, загружается немедленно, но выполнение скрипта не будет зависеть от порядка добавления его на страницу.

<!DOCTYPE html>
<html>
<head>
<title>ExampleHTMLPage</title>
  <script type="text/javascript" async src="singular1.js"></script>
  <script type="text/javascript" async src="singular2.js"></script>
   </head>
<body>
  <!--контент-->
</body>
</html>

<!DOCTYPE html>
<html>
<head>
  <title>Тег "noscript"</title>
  <script type="text/javascript" defer="defer" src="examplel.js"></script>
  <script type="text/javascript" defer="defer" src="example2.js"></script>
</head>
<body>
  <noscript>
    <p>Ваш браузер не поддерживает скрипты, либо они отключены!</p>
  </noscript>
</body>
</html>

Тег <noscript>

Содержимое тега <noscript> будет показано только если браузер не поддерживает выполнение сценариев или в браузере отключены скрипты
